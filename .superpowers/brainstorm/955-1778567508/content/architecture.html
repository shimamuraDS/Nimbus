整体架构方案

C++ & QML 天气桌面应用 - 选择你认为最合适的技术架构

A

Qt Quick + C++ Backend（推荐）

QML 负责全部 UI（系统托盘弹出面板、页面导航、动画），C++ 通过 QQmlApplicationEngine 暴露数据模型和业务逻辑。天气数据通过 C++ 的 QNetworkAccessManager 异步获取，使用 QSystemTrayIcon 管理系统托盘。

Pros

  • UI 与逻辑完全分离，QML 负责呈现，C++ 负责数据
  • Qt 原生支持系统托盘和通知
  • 跨平台潜力（Windows/Linux/macOS）
  • QML 声明式 UI 适合多页面导航

Cons

  • 学习曲线稍陡（QML + C++ 绑定）
  • 打包体积较大（~40MB+）
B

Qt Widgets + QML 混合

主窗口使用 QWidget 管理，嵌入 QQuickWidget 承载部分 QML 界面。使用 Windows API（Shell_NotifyIcon）直接管理系统托盘。

Pros

  • Widget 体系成熟稳定
  • 更精确的窗口位置控制

Cons

  • Widget + QML 混合增加复杂度
  • 代码风格不统一
  • 动画和主题能力受限
C

纯 C++ 原生 Windows

使用 Win32 API + Direct2D 渲染，不依赖 Qt。用 Windows.UI.Notifications API 实现通知。

Pros

  • 最小打包体积（~5MB）
  • 最深 Windows 集成

Cons

  • 开发效率低，UI 编写繁琐
  • 无法跨平台
  • 与 idea.md 指定的 QML 矛盾
