# 需求说明文档

## 1. 核心功能需求

### 1.1 天气查看
*   **当日天气查询**：首页以横向时间线展示未来 24 小时的逐小时天气和气温，当前小时自动高亮。
*   **历史天气**：支持向左导航查看"过去 7 天"的早晚天气与气温（基于本地逐小时缓存归档获取，每天取 08:00 为"早"，20:00 为"晚"）。注：逐小时 API 不返回湿度字段，因此历史天气不含湿度数据。
*   **未来预报**：支持向右导航查看"未来 7 天"的早晚天气信息（来自 API 的 future 接口）。

### 1.2 智能定位系统
*   **自动 IP 定位**：默认通过腾讯 IP 定位 API 获取当前城市，adcode 自动归一化到市级精度。
*   **手动定位**：用户点击"定位不准？"后可手动选择城市（覆盖全国 98 个城市），支持通过"返回自动定位"链接切回自动模式。

### 1.3 开机自启动
*   应用默认开启开机自启动，通过写入 Windows 注册表 `HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run` 实现。
*   用户可在设置页面通过开关控件自行启用或禁用开机自启动。

### 1.4 异常天气提醒
*   用户可自行配置多个"提醒时间点"（如 08:00, 20:30）。
*   **无提醒时间时**：系统默认检测未来1小时内的逐小时预报，当出现非晴天且非多云的天气（如阴、雨、雪、雾、沙尘等）时自动弹出 Windows 气泡通知。
*   **设置提醒时间后**：在时间命中时，不论晴天与否均触发提醒，结合腾讯 API 返回的天气描述或官方灾害预警，弹出 Windows 气泡通知。通知内容同时包含当前天气和即将出现的天气。
*   **双重判定机制**：
    1. 官方预警命中（`type=now&added_fields=alarm` 返回的 alarms 数组非空）
    2. 逐小时天气命中（无提醒时间时检测非晴非多云，有提醒时间时检测所有天气）

## 2. 非功能需求

*   **交互规范**：系统启动即静默驻留于任务栏托盘，不侵占任务栏窗口。界面唤出后，总面积限制在不超过屏幕总体分辨率的 1/12。
*   **缓存机制**：程序需有效整合逐小时预报及本地缓存，按 1 小时粒度存储天气数据，解决无直接官方历史查询 API 的短板。天气数据在应用启动与定位变更时统一刷新。
*   **性能要求**：应用内存占用 < 100MB，CPU 使用率 < 5%（空闲时）。
*   **可靠性**：断网时能正常加载本地缓存展示，并给予界面提示。

## 3. UI 交互流程

### 3.1 页面路由
*   中心节点为【当日天气主页】，顶部悬挂 Toolbar。
*   今日页点击【左侧 ◀ 按键】进入【过去天气页】（页面从左往右滑）；点击【右侧 ▶ 按键】进入【未来天气页】（页面从右往左滑）。
*   过去→今日：页面从右往左滑回；未来→今日：页面从左往右滑回。
*   设置页面采用垂直过渡动画（打开时从顶部向下滑入，返回时从下向上滑出）。

### 3.2 设置页面
*   Toolbar 右侧"设置"按键进入后，按键文本变更为"返回"；右侧同时有"−"最小化按键隐藏窗口到托盘。
*   设置页面采用上下可滚动布局（Flickable），内容超出时自动出现滚动条。
*   **开机自启动模块**：
    *   页面顶部显示"开机自启动"开关控件（Toggle Switch），用户可随时启用或禁用。
    *   开关采用电光青色强调色，切换时带有平滑的颜色和位移动画。
*   **定位模块**：
    *   自动定位模式：显示 "自动定位：【城市名】" + 可点击的 "定位不准？" 链接。
    *   手动定位模式：显示 "选择城市：" 标签 + 城市选择按钮（点击弹出省份/城市两列面板，共 98 个城市）+ "返回自动定位" 链接。
*   **提醒模块**：呈现一个可滚动的时间点列表。
    *   每个提醒时间点旁边显示对应的天气查看时长（如"未来30分钟"）。
    *   第一行：小时/分钟两个 ComboBox 下拉选择器，用于设置提醒时间点。
    *   第二行：时长小时/时长分钟两个 ComboBox，用于设置在提醒时间点之后查看天气的时间范围（0~23小时 + 0~59分钟），附带"时长"标签。
    *   第三行："添加"按钮（修改模式下变为"保存"+"取消"）。
    *   列表中每个时间点末尾附带"修改"按钮（蓝色文字）和"删除"按钮（红色文字）。
    *   点击"修改"后，时间选择器和时长选择器自动填充当前值，进入修改模式。

## 4. 界面尺寸规格

*   窗口宽 = 屏幕宽度 / 4
*   窗口高 = 屏幕高度 / 3
*   窗口面积 = 屏幕面积 / 12
*   窗口定位：屏幕右下角，距右边界 10px，距下边界 10px
*   无边框窗口，始终置顶
*   视觉风格：深色玻璃态（Dark Glassmorphism），半透明卡片叠加在深色渐变背景上
*   **Toolbar**：左侧"天气提醒助手"标题，右侧排列"设置/返回"按钮和"−"最小化按钮

## 5. 视觉设计规范

*   **配色方案**：暗色赛博朋克主题 (Dark Cyberpunk)
    *   背景渐变：`#111124` → `#06060c`（更深邃的暗色）
    *   玻璃态卡片背景：半透明白色叠加 `rgba(255,255,255,0.04~0.09)`
    *   强调色主色：电光青色 `#00f0ff`（accent）
    *   暖色强调：日落珊瑚 `#ff7b90`（accentWarm，用于过去页）
    *   辅助强调：柔和紫 `#a78bfa`（accentSecondary）
    *   发光边框：青色辉光 `rgba(0,240,255,0.19)`，珊瑚辉光 `rgba(255,123,144,0.19)`
    *   主文字：`#f8fafc`，次要文字：`#cbd5e1`，弱化文字：`#64748b`
*   **全局主题**：`Theme.qml` 集中管理所有视觉常量（颜色、间距、圆角、字体）
*   **动画**：StackView 页面切换带方向感知的滑动+淡入淡出效果（300ms, OutQuint 缓动）
*   **卡片变体**：`WeatherCard.isPast` 控制暖珊瑚/冷青色背景色切换
*   **图标**：全新高分辨率天气图标集（晴、多云、阴、雨、雪、雷阵雨、雾、沙尘、未知）
