# 需求说明文档

## 1. 核心功能需求

### 1.1 天气查看
*   **当日天气查询**：首页能够呈现当地当日早、晚的天气状况、气温及湿度参数。
*   **历史天气**：支持向左导航查看"过去 7 天"的早晚天气与气温（基于本地逐小时缓存归档获取，每天取 08:00 为"早"，20:00 为"晚"）。注：逐小时 API 不返回湿度字段，因此历史天气不含湿度数据。
*   **未来预报**：支持向右导航查看"未来 7 天"的早晚天气信息（来自 API 的 future 接口）。

### 1.2 智能定位系统
*   **自动 IP 定位**：默认通过腾讯 IP 定位 API 获取当前城市，adcode 自动归一化到市级精度。
*   **手动定位**：用户点击"定位不准？"后可手动选择城市（覆盖全国 98 个城市），支持通过"返回自动定位"链接切回自动模式。

### 1.3 异常天气提醒
*   用户可自行配置多个"提醒时间点"（如 08:00, 20:30）。
*   在时间命中时，结合腾讯 API 返回的恶劣天气描述（如暴雨、冰雹）或官方灾害预警，弹出 Windows 气泡通知。
*   **双重判定机制**：
    1. 官方预警命中（`type=now&added_fields=alarm` 返回的 alarms 数组非空）
    2. 逐小时恶劣天气命中（未来 3 小时内的逐小时预报包含极端天气关键字）

## 2. 非功能需求

*   **交互规范**：系统启动即静默驻留于任务栏托盘，不侵占任务栏窗口。界面唤出后，总面积限制在不超过屏幕总体分辨率的 1/12。
*   **缓存机制**：程序需有效整合逐小时预报及本地缓存，按 1 小时粒度存储天气数据，解决无直接官方历史查询 API 的短板。天气数据在应用启动与定位变更时统一刷新。
*   **性能要求**：应用内存占用 < 100MB，CPU 使用率 < 5%（空闲时）。
*   **可靠性**：断网时能正常加载本地缓存展示，并给予界面提示。

## 3. UI 交互流程

### 3.1 页面路由
*   中心节点为【当日天气主页】，顶部悬挂 Toolbar。
*   主页点击【左侧 ◀ 按键】进入【过去天气页】（新页面从左侧滑入）；点击【右侧 ▶ 按键】进入【未来天气页】（新页面从右侧滑入）。
*   过去/未来页面均支持相应按键反向 Pop 回主页（原路滑出）。
*   设置页面采用垂直过渡动画（打开时从顶部向下滑入，返回时从下向上滑出）。

### 3.2 设置页面
*   Toolbar 右侧"设置"按键进入后，按键文本变更为"返回"。
*   **定位模块**：
    *   自动定位模式：显示 "自动定位：【城市名】" + 可点击的 "定位不准？" 链接。
    *   手动定位模式：显示 "选择城市：" 标签 + 城市下拉框（98 个城市）+ "返回自动定位" 链接。
*   **提醒模块**：呈现一个可滚动的时间点列表。
    *   尾部拥有常驻按钮"添加时间点"。
    *   列表中已有时间点末尾附带"删除"按钮（红色文字）。

## 4. 界面尺寸规格

*   窗口宽 = 屏幕宽度 / 4
*   窗口高 = 屏幕高度 / 3
*   窗口面积 = 屏幕面积 / 12
*   窗口定位：屏幕右下角，距右边界 10px，距下边界 10px
*   无边框窗口，始终置顶
*   视觉风格：深色玻璃态（Dark Glassmorphism），半透明卡片叠加在深色渐变背景上

## 5. 视觉设计规范

*   **配色方案**：暗色主题
    *   背景渐变：`#1e1e2e` → `#11111b`
    *   当日/未来卡片背景：冷蓝半透明 `rgba(137,180,250,0.13)`
    *   过去卡片背景：暖金半透明 `rgba(255,213,153,0.15)`
    *   强调色：`#89b4fa`（蓝）、`#fab387`（暖橙）
    *   主文字：`#cdd6f4`，次要文字：`#a6adc8`
*   **全局主题**：`Theme.qml` 集中管理所有视觉常量
*   **动画**：StackView 页面切换带方向感知的滑动+淡入淡出效果
*   **卡片变体**：`WeatherCard.isPast` 控制暖金/冷蓝背景色切换
